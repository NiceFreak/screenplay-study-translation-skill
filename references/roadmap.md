# Roadmap & Cost Notes

A strategic anchor for this project. Not an AI runtime spec (those live in
`AI_AGENT_CONTRACT.md` / `AI_AGENT_PROJECT_SPEC.md`); this is the human owner's
direction, priorities, and cost playbook.

## 北极星 (North star)

> 看完一部喜欢的电影，想读它的剧本，但英语不足以读原版 —— 所以要一份
> **能独立读、读得舒服的中文剧本**。看片对照（成片与剧本的删改）是**增益**，
> 不是起点。

设计原则：**译文独立成立**（不看片也读得顺）；**对照成片更佳**（差异作为叠加层）。
当前渲染已支持这一双模 —— 无字幕即干净中文剧本，有字幕则浮出删改地图。

## 现状判断 (Status, 2026-06)

- **译文质量：已由真实通读验证**（I Saw the TV Glow 全本、Sinners 半本，挑不出
  毛病）。项目最初的目标已达成。
- **阅读体验（HTML/EPUB）**：经数次 PR 后已令人满意。
- **删改对照层**：功能完整，**冻结** —— 除非真实使用暴露需求，不再加功能。
- **没有走偏**：译文过关后推进到阅读体验，是 solved → next frontier 的正常推进。

## 成本两类，分开看 (Two cost buckets)

| 桶 | 是什么 | 性质 |
|---|---|---|
| **开发成本** | 改代码 / 改需求 / 返工 | **一次性**，摊薄到未来每部翻译；大部分已花完，不是复发账单 |
| **翻译成本** | 模型翻译一部剧本的 token | **每部复发** —— 唯一值得压的 |

关键结论：**渲染 / 阅读体验改动属于"开发成本"，建一次永久复用，对翻译成本零增量。**
EPUB 排版、导航、对照形态、删改呈现 —— 全是确定性脚本，不烧翻译 token。

历史账无法拆分（开发/讨论/实验/翻译混在一起，且未分开记），视为沉没的"建工具 +
学费"。往后只需一条纪律即可拿到干净数字（见下）。

## (a) 干净翻译成本测量清单 (run once)

目标：拿到**单部剧本的真实翻译成本**，不被开发/返工污染。

1. **开一个全新会话，只做翻译** —— 不改代码、不讨论需求。
2. 事先备好输入：剧本 PDF + 中英双语字幕 + 中文片名。
3. 走标准流程：`生成预览` → 确认风格 → `预览通过，继续跑完整本` →
   `生成成品 HTML` → `导出 EPUB`。
4. 期间**不夹带代码改动**；若发现 bug，记下来，**另开开发会话**再修。
5. 记录该会话的 token 用量 / 花费（Claude Code `/cost`，或 Anthropic Console 按
   日期/workspace 看 Usage）。
6. 这个数 = 这部剧本的真实翻译成本；除以页数得每页成本。

注意：`cost_report.py` 是**产物体积代理**，不是账单，别当真值。

## (b) 翻译成本杠杆 + 当前状态 (levers)

| 杠杆 | 作用 | 当前状态 |
|---|---|---|
| 匹配对白复用双语字幕中文 | 模型复制而非新造 → 输出更少 | ✅ 已用（策略） |
| 紧凑批次上下文 `package_batch_context.py` | 只喂当前页范围，不读全量 | ✅ 已用 |
| 5–10 页一批 | 摊薄固定开销 | ✅ 已用（配置） |
| prompt caching（skill/契约/术语复用） | 削减重复输入成本 | ⚙️ **Harness 层**（由 Claude Code 决定，技能本身不控制） |
| 模型分级（便宜跑量 + 强模型把关） | 降低跑量 $/token | ❌ 未用（质量已在当前模型验证，分级有一致性风险） |

结论：**token 杠杆大多已经在用。** 最大的未启用项是 prompt caching，那是 harness
设置而非代码改动；模型分级是唯一剩下的大杠杆，但有质量/一致性风险，你已选择不冒。
很可能**测一次就会发现"一部也就几美元"，不值得再优化**。
