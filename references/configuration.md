# Configuration

Project configuration should live in `project.yaml`. Keep it explicit and portable.

## Minimal Shape

```yaml
project:
  title: Example Film
  chinese_title: 示例影片
  source_language: en
  target_language: zh-CN

inputs:
  screenplay_pdf: path/to/screenplay.pdf
  subtitles: null

outputs:
  work_dir: work
  dist_dir: dist
  marker_inventory: work/source-markers.json
  subtitles_json: work/subtitles.json
  html: dist/screenplay-study.html
  pdf: null

page_mapping:
  title_pages: 1
  first_screenplay_pdf_page: 1
  displayed_page_offset: -1

translation:
  batch_size_pages: 10
  subtitle_labels: auto

audit:
  require_subtitles: false
  require_structured_markers: true
  paper_size: A4
  fail_on_missing_required_files: true
```

## Chinese Title

`project.chinese_title` is required for new translation projects. It is the
reader-facing Chinese film title used on the HTML cover and should be supplied
by the user when the project is initialized.

Keep `project.title` as the source/original title. Do not infer the Chinese
title from subtitles, filenames, or AI judgment when the user has not provided
one.

## Reference Subtitles

`inputs.subtitles` may be `null`, but high-quality Chinese subtitles are
strongly recommended. The user judges subtitle quality; the skill does not
automatically validate it.

When subtitles are present:

- dialogue translation uses subtitle content directly
- subtitle style becomes the main calibration input for `style-profile.json`
- non-dialogue elements are translated by AI using the style profile

When subtitles are absent:

- do not emit subtitle labels
- skip subtitle alignment checks
- AI translates all elements and output quality depends on model judgment

## Reader Notes

Project-specific reader notes should live in `references/reader_notes.md` next
to `project.yaml`. This file is the source of truth for the HTML reading note,
including edition-wide style conventions and professional format terms that
appear in the screenplay.

The HTML renderer may display this artifact, but must not infer professional
terms by scanning the current batch. If the artifact is absent, the renderer may
use only a minimal generic fallback note.

## Front Matter

Project-specific cover and title-page translations should live in
`references/front_matter.md` next to `project.yaml`. This file is the source of
truth for reader-facing title-page metadata, including title, writers, source
material, draft dates, revision-color labels, contact blocks, and other
cover-page information that should appear on the HTML cover.

The renderer may display this artifact, but must not infer title-page semantics
from raw PDF rows. If the artifact is absent, the renderer may use only a
minimal fallback title from the source title page and project config.

## Page Mapping

Keep page mapping explicit. Do not hide title-page offsets in code.

`displayed_page_offset` is added to the physical PDF page number to compute the displayed screenplay page when no richer mapping is available.

## Audit Configuration

The audit should fail missing required files, but warn rather than fail optional subtitle absence unless `require_subtitles` is true.

`outputs.marker_inventory` should point to `source-markers.json`, generated by `scan_markers.py`.

`outputs.subtitles_json` should point to the normalized subtitle events generated by `parse_subtitles.py` when `inputs.subtitles` is configured.

`outputs.html` is the default reader/review output. `outputs.pdf` should normally be `null` in v0.1. If PDF output is deliberately reopened later, it should be a reflowed reading/print edition with source labels such as "原剧本第 X 页", not a page-aligned comparison PDF.

`audit.require_structured_markers` should stay true for new projects. It requires generated HTML to mark screenplay-body structure with machine-readable marker attributes, not only translated text.
